## 포함 기능

### 기능 1. 수신 정책 및 승인 UX

- 자동 수신, 수동 승인, 허용 사용자만 자동 수신 정책 구현
- 저장 폴더 지정 및 정책별 분기 처리
- 수신 요청 팝업과 승인/거절 액션

### 기능 2. 전송 이력 화면

- 발신/수신 이력 조회
- 실패 원인, 전송 시간, 상대 노드, 파일 크기 표시
- 완료 파일 경로 열기 또는 복사 동작

### 기능 3. 로그/설정 화면

- 로그 레벨 설정
- 포트, 기본 저장 경로, 동시 전송 수, 디스커버리 주기 설정
- 최근 오류 로그 조회 또는 디버그 로그 내보내기

## 구현 체크리스트

- [ ] 수신 정책이 실제 수신 파이프라인에 반영된다.
- [ ] 승인 필요 정책에서 사용자 승인 전 파일 저장이 시작되지 않는다.
- [ ] 전송 이력이 DB에 저장되고 목록으로 조회된다.
- [ ] 설정 변경이 앱 재시작 후에도 유지된다.
- [ ] 로그 뷰어 또는 로그 내보내기 흐름이 동작한다.
- [ ] 실패 사유가 사용자용 메시지와 내부 코드 모두로 확인 가능하다.

## 검증

- [ ] 자동 수신과 수동 승인 정책이 기대대로 다르게 동작하는지 확인한다.
- [ ] 실패한 전송이 이력 화면에서 원인과 함께 조회되는지 확인한다.
- [ ] 로그 레벨을 바꾼 뒤 저장되는 로그 양이 달라지는지 확인한다.
- [ ] 저장 경로를 변경한 뒤 새 수신 파일이 새 경로로 저장되는지 확인한다.

## 완료 기준

- 수신 정책과 설정을 손으로 수정하지 않고 UI에서 관리할 수 있다.
